# BUG-016: C# parser not registered in NewDefaultRegistry

**Task:** TASK-016 — C# Tree-Sitter Grammar
**Author:** bravo-1
**Date:** 2026-03-15

## What's Missing

The C# parser was implemented and tests pass, but `NewCSharpParser()` was not added to `NewDefaultRegistry()` in `internal/parser/registry.go`. This means `engram index` will not index .cs files by default.

## How to Fix

Add one line to `NewDefaultRegistry()` in `internal/parser/registry.go`:
```go
r.Register(NewCSharpParser())
```

Add it after the existing Java parser registration.

## Impact

Low — C# files won't be indexed until this is fixed. All other languages work.
