# BUG-016: C#, Ruby, PHP parsers not registered in NewDefaultRegistry

**Tasks:** TASK-016, TASK-021
**Date:** 2026-03-15

## What's Missing

Three parsers (C#, Ruby, PHP) were implemented and tests pass, but they were not added to `NewDefaultRegistry()` in `internal/parser/registry.go`. `engram index` will not index .cs/.rb/.php files by default.

## How to Fix

Add three lines to `NewDefaultRegistry()` in `internal/parser/registry.go`:
```go
r.Register(NewCSharpParser())
r.Register(NewRubyParser())
r.Register(NewPHPParser())
```

## Impact

Low — .cs/.rb/.php files won't be indexed until this is fixed. Other 6 languages work fine.
