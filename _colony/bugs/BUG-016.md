# BUG-016: Multiple parsers not registered in NewDefaultRegistry

**Tasks:** TASK-016, TASK-021, TASK-027
**Date:** 2026-03-15

## What's Missing

Five parsers (C#, Ruby, PHP, C, C++) were implemented and tests pass, but they were not added to `NewDefaultRegistry()` in `internal/parser/registry.go`. `engram index` will not index these file types by default.

Note: Swift and Kotlin (TASK-026) ARE registered correctly.

## How to Fix

Add five lines to `NewDefaultRegistry()` in `internal/parser/registry.go`:
```go
r.Register(NewCSharpParser())
r.Register(NewRubyParser())
r.Register(NewPHPParser())
r.Register(NewCParser())
r.Register(NewCPPParser())
```

## Impact

Low — .cs/.rb/.php/.c/.cpp/.h files won't be indexed until this is fixed. 8 languages (Go, Python, TS, JS, Rust, Java, Swift, Kotlin) work fine.
