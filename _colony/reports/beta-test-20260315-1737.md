## P0 Issue: FTS5 Build Tag Ship-Blocker

**Severity:** P0 — ship-blocker
**Status:** UNRESOLVED — CEO flagged in directive, no task exists to fix it

Any user running `go install` or `go test ./...` without the build tag will hit 99 test failures and a broken binary. This affects:
- `go install github.com/TomOst-Sec/colony-project/cmd/engram@latest`
- CI/CD systems that don't use the Makefile
- Contributors running tests for the first time

**Fix:** Add a Go file (e.g., `internal/storage/cgo_flags.go`) with `// #cgo CFLAGS: -DSQLITE_ENABLE_FTS5` to enable FTS5 unconditionally when CGo is active, eliminating the need for the build tag.

**Bug task filed:** TASK-036 (P0, alpha)

## P1 Issue: Parser Registry Incomplete (BUG-016 partially unresolved)

**Severity:** P1
**Status:** PARTIALLY FIXED — CSharp was registered in TASK-022, but 6 parsers remain unregistered

Missing from `NewDefaultRegistry()` in `internal/parser/registry.go`:
- `NewRubyParser()`
- `NewPHPParser()`
- `NewCParser()`
- `NewCPPParser()`
- `NewLuaParser()`
- `NewZigParser()`

These parsers all exist and their tests pass, but they won't be used during indexing because they're not in the default registry. This means `.rb`, `.php`, `.c`, `.cpp`, `.h`, `.lua`, `.zig` files are silently skipped.

**Bug task filed:** TASK-037 (P1, bravo)
